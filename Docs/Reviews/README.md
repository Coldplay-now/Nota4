## 🔍 审查类型说明

### 代码质量审查
- 检查代码可读性、可维护性
- 评估设计模式使用
- 检查命名规范

### 安全审查
- 检查安全漏洞
- 评估数据保护措施
- 审查权限控制

### 性能审查
- 识别性能瓶颈
- 评估资源使用
- 检查内存泄漏

### 架构审查
- 评估架构合理性
- 检查模块划分
- 审查依赖关系

### 穷尽性审查
- 检查 switch 语句完整性
- 评估错误处理覆盖度
- 审查状态转换完整性

---

## 📅 审查计划

### 定期审查
- **周审查**: 每周五进行代码质量审查
- **月审查**: 每月最后一周进行全面审查
- **季度审查**: 季度末进行架构审查

### 触发审查
- 重大功能开发完成后
- 重构任务完成后
- 发现严重 Bug 后
- 性能问题报告后
