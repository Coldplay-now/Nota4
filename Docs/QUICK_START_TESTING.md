## ✅ 前置条件

- [x] Xcode 已安装
- [x] 项目已在 Xcode 中打开
- [x] 脚本已有执行权限

---

## 🚀 3 步开始测试

### 步骤 1: 启动日志监控（1 分钟）

打开终端，运行：

```bash
cd /Users/xt/LXT/code/trae/1107-model-eval/Nota4
./Scripts/watch_app_logs.sh
```

您会看到：

```
╔════════════════════════════════════════╗
║   Nota4 实时日志监控                  ║
╔════════════════════════════════════════╗

📝 现在请在 Xcode 中点击 Run 按钮
🔍 我会自动捕获所有日志...

⏸  按 Ctrl+C 停止监控
```

**✅ 监控已就绪！**

---

### 步骤 2: 在 Xcode 中运行应用（1 分钟）

1. 在 Xcode 中，点击左上角的 **▶️ Run** 按钮（或按 `⌘R`）
2. 等待编译完成
3. 应用自动启动

**✅ 应用已启动！**

---

### 步骤 3: 交互测试（5-10 分钟）

参考 [测试清单](./MANUAL_TEST_CHECKLIST.md)，进行以下核心测试：

#### 必测功能（5 分钟）

1. **创建笔记** ✏️
   - 点击"新建笔记"按钮
   - 输入标题和内容
   - 观察自动保存

2. **编辑笔记** 📝
   - 选择一条笔记
   - 修改内容
   - 切换到其他笔记
   - 返回查看是否已保存

3. **搜索功能** 🔍
   - 在搜索框输入关键词
   - 验证搜索结果
   - 清空搜索

4. **标签功能** 🏷️
   - 点击侧边栏的标签
   - 查看筛选结果

5. **星标功能** ⭐
   - 星标一条笔记
   - 在"星标笔记"分类查看

6. **删除和恢复** 🗑️
   - 删除一条笔记
   - 在"回收站"查看
   - 恢复笔记

#### 观察终端输出

在测试过程中，观察终端的日志输出：

- 🟢 **绿色**: 正常日志（如"笔记已创建"）
- 🟡 **黄色**: 警告（如"查询较慢"）
- 🔴 **红色**: 错误（如"导入失败"）

**如果看到红色错误，立即记录下来！**

---

### 步骤 4: 停止监控并查看结果（1 分钟）

1. 回到终端窗口
2. 按 `Ctrl+C` 停止监控
3. 查看统计摘要：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📊 监控统计:
  错误: 0
  警告: 2
  信息: 156

✅ 日志已保存到:
  Logs/app_console_20251116_103045.log
```

## 📊 结果判断

### ✅ 测试通过

如果满足以下条件：
- 错误数 = 0
- 应用无崩溃
- 核心功能正常工作

**🎉 恭喜！可以进入发布流程了！**

---

### ⚠️ 有小问题

如果有少量警告（< 5 个）但功能正常：
- 记录警告内容
- 评估是否影响用户体验
- 决定是否修复后再发布

**查看警告详情**：

```bash
cat Logs/app_console_*.log | grep WARN
```

---

### ❌ 有严重问题

如果发现以下情况：
- 应用崩溃
- 核心功能无法使用
- 错误数 > 0

**查看错误详情**：

```bash
cat Logs/app_console_*.log | grep ERROR
```

**下一步**：
1. 记录错误信息
2. 定位问题代码
3. 修复问题
4. 重新测试

## 🐛 常见问题

### Q: 终端没有日志输出

**A**: 
1. 确保应用在运行
2. 在应用中进行一些操作
3. 检查 `Logs/` 目录是否有文件生成

### Q: 应用无法启动

**A**:
1. 检查 Xcode 的编译错误
2. 运行 `swift build` 查看错误
3. 查看 [故障排除指南](./TROUBLESHOOTING.md)

### Q: 看到很多警告

**A**:
1. 不是所有警告都是问题
2. 重点关注"ERROR"和"FATAL"
3. 查看 [调试监控指南](./DEBUG_MONITORING_GUIDE.md) 了解如何分析
